This guide covers building a backend API with Express.js and making HTTP requests from the frontend using Axios. You'll learn the fundamentals of client-server communication.

üöÄ What is Express.js?

Express.js is a minimal and flexible Node.js web framework that makes it easy to build web applications and APIs. It handles HTTP requests and responses.

Key Concepts:

  • Server: A computer that listens for incoming requests
  • Port: A virtual "door" where the server listens (e.g., 3000)
  • Routes: URLs that handle specific HTTP methods (GET, POST, PUT, DELETE)
  • Middleware: Functions that process requests before they reach routes

Basic Structure:

const express = require("express")
const app = express()

app.listen(3000, () => {
    console.log("Serving on port 3000")
})

üåê HTTP Methods (CRUD Operations)

Method Purpose Description
GET Read Retrieve data from the server
POST Create Send new data to the server
PUT Update Update existing data on the server
DELETE Delete Remove data from the server

‚öôÔ∏è Backend: Express.js Server

The server handles incoming requests and sends responses back to the client.

Complete Example (app.js):

const express = require("express")
const app = express()
const cors = require("cors")

app.use(cors())    // Enable CORS

let users_data = [
    {name: "Karan"},
    {name: "Divya"},
]

// GET - Retrieve users
app.get("/users", (req, res) => {
    res.send(users_data)
})

// POST - Create new user
app.post("/users", (req, res) => {
    res.send("Users Post")
})

// PUT - Update user
app.put("/users", (req, res) => {
    res.send("Users Put")
})

// DELETE - Remove user
app.delete("/users", (req, res) => {
    res.send("Users Delete")
})

app.listen(3000, () => {
    console.log("Serving on port 3000")
})
üí° Note on CORS: CORS (Cross-Origin Resource Sharing) allows frontend and backend to communicate even if they're on different ports or domains. Without it, browser security blocks the requests.

üíª Frontend: Axios HTTP Requests

The frontend uses Axios to communicate with the backend server. Axios is a library that simplifies making HTTP requests.

Setup in HTML:

<script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

Making HTTP Requests:

// GET - Fetch data
axios.get('http://localhost:3000/users')
    .then(res => {
        console.table(res.data)
    })

// POST - Send data
axios.post('http://localhost:3000/users')
    .then(res => {
        console.log("FE POST: ", res.data)
    })

// PUT - Update data
axios.put('http://localhost:3000/users')
    .then(res => {
        console.log("FE PUT: ", res.data)
    })

// DELETE - Remove data
axios.delete('http://localhost:3000/users')
    .then(res => {
        console.log("FE DELETE: ", res.data)
    })
üí° Note: Each request uses .then() to handle the response. This is a Promise pattern that waits for the server to respond before executing the next line.

üìñ Key Terminology

  • Localhost: Your own computer (127.0.0.1)
  • Port 3000: The address where the server listens
  • Request: Data sent from client to server
  • Response: Data sent from server back to client
  • Endpoint: A specific URL route (e.g., /users)
  • Promise: A JavaScript object that handles async operations
  • .then(): Executes code after a Promise resolves

üèÉ How to Run on Local System

Follow these steps to run both the backend and frontend on your computer using VS Code.

Step 1: Install Dependencies

Open terminal in VS Code and run:

npm install express cors
npm install --save-dev nodemon

Step 2: Create Project Structure

project-folder/

Step 3: Run Backend Server

Option 1 - Without Hot Reload:

npm start
// or
node app.js

Option 2 - With Hot Reload (Nodemon):

nodemon app.js
üí° What is Nodemon? Nodemon automatically restarts your server whenever you make changes to the code. This saves you from manually restarting the server every time. Much better for development!

Step 5: Run Frontend with Live Server

In a new terminal, install Live Server extension first:

  • Go to VS Code Extensions (Ctrl+Shift+X / Cmd+Shift+X)
  • Search for "Live Server"
  • Install by Ritwick Dey

Then right-click on index.html and select "Open with Live Server"

Step 5: Verify Everything Works

  • Backend should be running on: http://localhost:3000
  • Frontend should be running on: http://localhost:5500 (or similar)
  • Open browser console (F12) to see API responses
üí° Pro Tips:
  • Keep two terminals open: one for nodemon app.js, one for frontend
  • Use nodemon app.js during development for auto-restart
  • Live Server auto-refreshes your HTML when you save
  • Use browser DevTools (F12) to debug API calls

Troubleshooting

  • Port 3000 already in use? Change port in app.js: app.listen(3001, ...)
  • CORS errors? Make sure CORS middleware is enabled in app.js
  • Axios not working? Check that Axios CDN link is in your HTML
  • nodemon not found? Run npm install --save-dev nodemon again
