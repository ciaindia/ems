HTTP Methods

HTTP methods (verbs) define the action to be performed on a resource.

GET
Safe Idempotent

Retrieves data from the server without modifying anything. Can be cached and bookmarked.

GET /api/users/123
POST

Submits data to create a new resource. Not idempotent - multiple calls create multiple resources.

POST /api/users
Body: {"name": "John", "email": "john@example.com"}
PUT
Idempotent

Updates or replaces an entire resource. Sends complete resource data.

PUT /api/users/123
Body: {"name": "John Doe", "email": "john@example.com", "age": 30}
PATCH

Partially updates a resource. Only sends changed fields.

PATCH /api/users/123
Body: {"email": "newemail@example.com"}
DELETE
Idempotent

Removes a resource from the server.

DELETE /api/users/123
HEAD
Safe Idempotent

Same as GET but returns only headers without the body. Useful for checking resource existence.

HEAD /api/users/123
OPTIONS
Safe Idempotent

Returns supported HTTP methods for a resource. Used in CORS preflight requests.

OPTIONS /api/users

Security Headers

Strict-Transport-Security (HSTS)
Security

Forces browsers to use HTTPS connections only.

Strict-Transport-Security: max-age=31536000; includeSubDomains
X-Content-Type-Options
Security

Prevents browsers from MIME-sniffing a response away from the declared content-type.

X-Content-Type-Options: nosniff
X-Frame-Options
Security

Protects against clickjacking attacks by controlling whether a page can be displayed in a frame.

X-Frame-Options: DENY
X-Frame-Options: SAMEORIGIN
Content-Security-Policy (CSP)
Security

Controls which resources can be loaded to prevent XSS and other attacks.

Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline'
X-XSS-Protection
Security

Legacy header to enable XSS filtering (mostly deprecated in favor of CSP).

X-XSS-Protection: 1; mode=block

Key Concepts

Idempotency

Multiple identical requests have the same effect as a single request. GET, PUT, DELETE, HEAD, and OPTIONS are idempotent. POST and PATCH are not idempotent.

Safety

A safe method doesn't modify server state. GET, HEAD, and OPTIONS are safe methods - they only retrieve information without causing side effects.

CORS (Cross-Origin Resource Sharing)

A mechanism that uses HTTP headers to allow a web application running at one origin to access resources from a different origin. Key headers include Access-Control-Allow-Origin, Access-Control-Allow-Methods, and Access-Control-Allow-Headers.

Custom Headers

Applications can define custom headers for specific purposes. Traditionally prefixed with "X-" (though this convention is deprecated), such as X-API-Key, X-Request-ID, or X-Custom-Data.

Status Codes

HTTP status codes indicate the result of a request:

‚Ä¢ 2xx: Success (200 OK, 201 Created)

‚Ä¢ 4xx: Client Error (400 Bad Request, 401 Unauthorized, 404 Not Found)
